

# שינוי סטטוס לקוח ישירות ממודול הנהלת חשבונות

## מה יהיה?
עמודת הסטטוס בטבלת הלקוחות במודול הנהלת חשבונות תהפוך לאינטראקטיבית - לחיצה על הבאדג' של הסטטוס תפתח דרופדאון שמאפשר לשנות את הסטטוס (פעיל / בקליטה / מושהה / סיים) ישירות מהטבלה, בלי לפתוח את דיאלוג עריכת הלקוח המלא.

השינוי יהיה דו-כיווני - עדכון הסטטוס מהנהלת חשבונות יעדכן את הלקוח בטבלת `clients`, ולכן השינוי יופיע גם בדף הלקוחות ובכל מקום אחר במערכת.

## איך זה יעבוד
- לחיצה על באדג' הסטטוס בשורת הלקוח תפתח Select dropdown קטן
- בחירת סטטוס חדש תעדכן ישירות את הלקוח בבסיס הנתונים
- כל ה-queries הרלוונטיים (clients, accounting-clients, finance-summary) יתרעננו אוטומטית
- לא נוצרים דיאלוגים כפולים - נשתמש באותו מנגנון שכבר קיים

## פרטים טכניים

### קובץ: `src/pages/AccountingIntegrations.tsx`

1. **הוספת mutation לעדכון סטטוס לקוח** - mutation חדש `updateClientStatus` שמעדכן את שדה `status` בטבלת `clients` ומרענן את כל ה-queries הרלוונטיים

2. **החלפת באדג' הסטטוס הסטטי בקומפוננטת Select אינטראקטיבית** - בשורה 939 (עמודת הסטטוס בטבלה), במקום `getStatusBadge(client.status)` תהיה Select dropdown עם הסטטוסים: פעיל, בקליטה, מושהה, סיים. ה-Select יקבל סטייל צבעוני לפי הסטטוס הנוכחי (ירוק לפעיל, כתום לבקליטה, אפור למושהה, אדום לסיים)

3. **Invalidation של queries** - בהצלחה, נרענן: `accounting-clients`, `clients`, `finance-summary` כדי להבטיח סנכרון מלא
