## מטרה
החלפת הצגת הדוחות/דשבורדים בטאב "דוחות" מ-Collapsible נערמים — למיני־טאבים אופקיים, ליד כפתור "ניהול שיוכים", כדי לעבור בקלות בין דוח לדוח בלי גלילה.

## שינויים — `src/components/clients/ClientTablesTab.tsx`

1. **איחוד פריטים לרשימה אחת**: יצירת `items` שמשלב dashboards + tables לפי הסדר הקיים (קודם דשבורדים, אחר כך טבלאות), כל אחד עם `key`, `label`, `icon` (LayoutDashboard / renderTableIcon), ו־`type`.

2. **state חדש**: `activeTabId` שמופעל אוטומטית לפריט הראשון; אם הפריט הפעיל נמחק/הוסר — להחליף ל־first item.

3. **שורת טאבים אופקית** (במקום הרשימה האנכית):
   - שימוש ב־`Tabs` של shadcn ב־`dir="rtl"`.
   - `TabsList` עם גלילה אופקית (`overflow-x-auto flex-nowrap`) כדי להכיל הרבה דוחות.
   - כל `TabsTrigger` מציג את האייקון של הדוח + שם מקוצר (`truncate max-w-[160px]`).
   - הטאבים יוצגו באותה שורה של "ניהול שיוכים" — flex container עם הכפתור בקצה אחד והטאבים בצד השני.

4. **תוכן הטאב הפעיל**:
   - דשבורד פעיל → `<ClientDashboardPanel ... />` בתוך מסגרת עם header קטן (אייקון פתיחה במסך מלא + הסרת שיוך).
   - טבלה פעילה → `<ClientReportPanel ... />` עם header דומה (פתיחה ב-route + הסרת שיוך + סטטוס חיבור).
   - הסרת ה־Collapsible לחלוטין; אין יותר accordion.

5. **שמירה על הקיים**:
   - חלק "ניהול שיוכים" (השייוך/דרופדאון לטבלה ולדשבורד) נשאר זהה — רק נפתח/נסגר ע"י אותו state.
   - מצב ריק (`!hasContent`) נשאר.
   - לוגיקת link/unlink, queries — ללא שינוי.

## הערות UI
- RTL נשמר.
- אייקון של פלטפורמה (Facebook/Google Ads/Analytics/SEO) מגיע אוטומטית דרך `renderTableIcon` לפי `integration_type` של הטבלה — כך שמיני־הטאבים מקבלים תיוג ויזואלי "פייסבוק/גוגל/SEO/דשבורד" ללא צורך בלוגיקה חדשה.
- אין שינויי backend, אין שינוי לקומפוננטות אחרות.
