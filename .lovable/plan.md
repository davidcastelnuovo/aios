

# תיקון תצוגת הכנסות חד פעמיות + הוספת הוצאה משויכת לספק

## בעיה 1: ההכנסה לא מופיעה
הרשומה קיימת בבסיס הנתונים (חודש `2026-02`), אבל החודש הנבחר כברירת מחדל הוא החודש הקודם (`2026-01`). לכן ההכנסה לא מוצגת. צריך לוודא שאחרי הוספת הכנסה, החודש הנבחר מתעדכן לחודש של ההכנסה שנוספה, כך שהמשתמש רואה אותה מיד.

## בעיה 2: הוספת הוצאה משויכת לספק
צריך להוסיף לטבלה `one_time_incomes` עמודות לתיעוד הוצאה משויכת - ספק וסכום הוצאה.

## שינויים

### 1. מיגרציה - הוספת עמודות לטבלה
הוספת העמודות הבאות ל-`one_time_incomes`:
- `supplier_id` (UUID, nullable, references suppliers)
- `expense_amount` (NUMERIC, default 0) - סכום ההוצאה לספק

### 2. עדכון הטופס בדיאלוג (`AccountingIntegrations.tsx`)
- הוספת שדה "ספק" (דרופדאון מרשימת הספקים הקיימת)
- הוספת שדה "סכום הוצאה" (מספרי)
- שני השדות אופציונליים

### 3. תיקון ניווט חודש
אחרי הוספת הכנסה חד פעמית, לעדכן את `selectedMonth` לחודש שנבחר בטופס כדי שההכנסה תופיע מיד.

### 4. עדכון טבלת התצוגה
הוספת עמודות "ספק" ו"הוצאה" לטבלה של הכנסות חד פעמיות, כדי שהמשתמש יראה את ההוצאה המשויכת.

### 5. עדכון ה-mutation
עדכון ה-`createOneTimeIncome` כדי לשלוח גם `supplier_id` ו-`expense_amount`.
