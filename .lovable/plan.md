

# שדרוג מודול חתימות דיגיטליות — הגדרת מיקום חתימה על מסמך

## מה קיים היום
- יוצר המסמך מקליד טקסט או מעלה קובץ
- מוסיף חותמים (שם + אימייל)
- שולח לחתימה → החותם מקבל קישור `/sign/:token`
- בעמוד החתימה: רואה את המסמך + חותם בקנבס נפרד למטה
- **אין אפשרות להגדיר איפה בדיוק החתימה מופיעה על המסמך**

## מה צריך לבנות

### 1. תצוגת מסמך עם הגדרת מיקום חתימה (יוצר המסמך)
- כשיוצר מסמך בטאב "העלאת מסמך" (PDF/תמונה), להציג תצוגה מקדימה של הקובץ
- יוצר המסמך לוחץ על המסמך כדי **לגרור ולמקם "שדה חתימה"** לכל חותם
- כל שדה חתימה = מלבן צבעוני עם שם החותם, ניתן לגרירה ושינוי גודל
- מיקום נשמר כ-JSON בטבלה (x, y, width, height, page)

### 2. שינוי DB — הוספת שדה `signature_position` ל-`signature_recipients`
```sql
ALTER TABLE signature_recipients 
ADD COLUMN signature_position jsonb DEFAULT null;
-- Format: { "x": 150, "y": 400, "width": 200, "height": 80, "page": 1 }
```

### 3. עמוד חתימה משודרג (`SignDocument.tsx`)
- במקום קנבס נפרד למטה, להציג את המסמך עם **אזור חתימה מסומן** במיקום שהוגדר
- החותם חותם ישירות באזור המסומן
- אם לא הוגדר מיקום — fallback להתנהגות הנוכחית (קנבס נפרד)

### 4. תמיכה בקישור למסמך קיים (URL)
- הוספת טאב שלישי "קישור למסמך" ביצירת מסמך
- המשתמש מדביק URL של מסמך (PDF, תמונה)
- המסמך מוצג באמצעות iframe/img ומאפשר הצבת שדות חתימה

## קבצים לשינוי/יצירה

| קובץ | שינוי |
|-------|-------|
| `signature_recipients` (DB) | הוספת עמודת `signature_position jsonb` |
| `src/pages/Signatures.tsx` | הוספת טאב "קישור למסמך", תצוגה מקדימה עם drag-to-place שדות חתימה |
| `src/pages/SignDocument.tsx` | הצגת אזור חתימה במיקום המוגדר על גבי המסמך |
| `src/components/signatures/DocumentPreview.tsx` | **חדש** — רכיב תצוגת מסמך עם שדות חתימה draggable |
| `src/components/signatures/SignatureFieldPlacer.tsx` | **חדש** — רכיב drag & drop למיקום שדה חתימה על המסמך |

## זרימת עבודה חדשה
```text
יוצר מסמך:
  1. בוחר "העלאת מסמך" או "קישור למסמך"
  2. מוסיף חותמים (שם + אימייל)
  3. רואה תצוגה מקדימה של המסמך
  4. גורר שדה חתימה לכל חותם למיקום הרצוי על המסמך
  5. שומר → שולח לחתימה

חותם:
  1. פותח קישור /sign/:token
  2. רואה את המסמך עם אזור חתימה מסומן
  3. חותם ישירות באזור המסומן
```
