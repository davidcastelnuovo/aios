

# החלת פילטרים על דשבורד CRM סוכנות

## הבעיה
הדשבורד מציג את כל הלקוחות של הטננט ללא הבחנה — אין סינון לפי סוכנות נבחרת, ואין הגבלת צפייה לקמפיינרים (שצריכים לראות רק לקוחות משויכים אליהם).

## הפתרון
ליישר את DMMDashboard.tsx עם אותו מודל הרשאות שקיים ב-Clients.tsx:

### 1. הוספת hooks חסרים
ייבוא `useAgency`, `useUserAgencies`, `useUserRole` לקומפוננטה.

### 2. פילטר סוכנות (agency dropdown)
- הוספת dropdown לבחירת סוכנות בשורת הפילטרים (כמו שקיים בדפים אחרים).
- שליפת רשימת סוכנויות (owned + shared via `agency_tenant_access`).
- סינון הלקוחות לפי הסוכנות הנבחרת ב-`useAgency`.

### 3. הגבלת צפייה לפי תפקיד
- **Owner / Team Manager / Super Admin** — רואים את כל הלקוחות (בסוכנות הנבחרת).
- **Campaigner** — רואים רק לקוחות שמשויכים אליהם דרך `client_team`.
- **SEO** — רואים רק לקוחות משויכים.

### 4. שינויים טכניים בקובץ `src/pages/DMMDashboard.tsx`

**שאילתת הלקוחות (שורה 133-153):**
- הוספת סינון `.eq("agency_id", selectedAgency)` כשנבחרה סוכנות ספציפית.
- הוספת `.or(...)` לסוכנויות משותפות (shared agencies) כשבחירה "הכל".

**סינון לפי קמפיינר (post-fetch):**
- שליפת `campaignerClientIds` מ-`client_team` (כמו ב-Clients.tsx).
- אם המשתמש הוא campaigner (לא team_manager/owner) — סינון הרשימה רק ללקוחות משויכים.

**UI — dropdown סוכנות:**
- הוספה לשורת הפילטרים בין הכותרת לשורת החיפוש.

### קבצים לעריכה
- `src/pages/DMMDashboard.tsx`

### תוצאה
- דשבורד CRM יסנן לפי סוכנות נבחרת (מהסלקטור הגלובלי).
- קמפיינרים יראו רק לקוחות משויכים אליהם.
- owners/team managers ימשיכו לראות הכל.
