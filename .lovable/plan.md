

# תיקון: ביטול קריאה אוטומטית ל-Ahrefs + בדיקת תצוגת דוח עימגה

## בעיות שזוהו

1. **קריאה אוטומטית ל-Ahrefs API בכל פתיחת דוח** — ב-`SeoDashboardView.tsx` שורה 178-184, יש `useEffect` שמפעיל `fetchComparisons` אוטומטית בכל פעם שהדשבורד נפתח. זה שולח 2 קריאות API ל-Ahrefs (3 חודשים + שנה), מה שצורך קרדיטים מיותרים.

2. **הדוח של עימגה** — הנתונים קיימים בבסיס הנתונים (4 דוחות, כולל `client_id` תקין). ייתכן שהקריאה האוטומטית נכשלת וזה גורם לבעיית תצוגה, או שיש בעיית RLS.

## תוכנית

### שלב 1: ביטול הקריאה האוטומטית ל-Ahrefs API
- **הסרת ה-`useEffect` האוטומטי** (שורות 178-184) שמפעיל `fetchComparisons` בכל פתיחת דוח
- **שמירת כפתור הסנכרון הידני** ("סנכרון Ahrefs") — כך המשתמש שולט מתי לקרוא ל-API
- נתוני 3 חודשים ושנה יוצגו **רק** כשהמשתמש לוחץ ידנית על "סנכרון Ahrefs"

### שלב 2: שמירת נתוני השוואה בבסיס הנתונים (Cache)
- כשהמשתמש לוחץ "סנכרון Ahrefs", נשמור את נתוני ההשוואה ב-`ahrefs_reports` כשדה נוסף (למשל `comparison_data`)
- בפתיחה הבאה, נטען את הנתונים מהבסיס נתונים במקום לקרוא ל-API
- כפתור "סנכרון Ahrefs" ירענן את הנתונים וישמור מחדש

### שלב 3: תיקון תצוגת דוח עימגה
- בדיקה שהדוח נטען כראוי ללא תלות בקריאה חיצונית
- וידוא שהדשבורד מציג את הנתונים מהדוח המקומי גם בלי enrichment חיצוני

## קבצים שישתנו
- `src/components/dynamic-tables/SeoDashboardView.tsx` — הסרת auto-fetch, טעינת cache
- `src/hooks/useAhrefsEnrichment.ts` — הוספת לוגיקת שמירה/טעינה מ-DB
- `supabase/functions/ahrefs-webhook/index.ts` — אופציונלי: הוספת שדה `comparison_data` בעדכון

## פרטים טכניים
- Migration: הוספת עמודה `comparison_data JSONB` לטבלת `ahrefs_reports` לשמירת cache של נתוני ההשוואה
- ה-cache יישמר עם timestamp כדי שהמשתמש ידע מתי הנתונים רוענו לאחרונה
- הדשבורד יטען קודם נתונים מקומיים, ורק בלחיצה ידנית יפנה ל-Ahrefs API
